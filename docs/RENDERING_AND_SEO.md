# Рендеринг и SEO

> **Важно:** проект собран на **Vite + React** (SPA), а не на Next.js.

## Текущая ситуация

| Вариант | У вас | Для поисковиков и ИИ |
|--------|--------|------------------------|
| ✅ SSR / SSG | ❌ Нет | Идеал — контент сразу в HTML |
| ⚠️ SPA через JS | ✅ Да | Риск — бот должен выполнять JS |
| ❌ Только клиентский рендер | ✅ Сейчас так | Плохо — без JS страница почти пустая |

**Итог:** основной контент (лендинг, текст) рисуется только после загрузки и выполнения JavaScript. Поисковики и ИИ-краулеры могут видеть только то, что есть в исходном HTML.

## Что уже сделано

1. **`index.html`** — добавлены:
   - мета `description` и `keywords`;
   - Open Graph (`og:title`, `og:description`, `og:locale`) для превью и краулеров;
   - блок `<noscript>` с заголовком и кратким текстом (виден без JS).
2. **`public/robots.txt`** и **`public/sitemap.xml`** — настроены для индексации.

Этого достаточно, чтобы в сниппетах и превью отображался осмысленный текст, но **полный лендинг по-прежнему только после выполнения JS**.

## Как улучшить (если нужен «идеал» для ИИ и SEO)

### Вариант A: Пререндер одной страницы (без перехода на Next.js)

Использовать плагин, который при сборке генерирует статический HTML для главной:

- **[vite-plugin-prerender](https://github.com/ElMassimo/vite-plugin-prerender)** или аналог: после `vite build` сохраняется HTML с уже отрендеренным лендингом.
- Плюсы: один `npm run build`, без смены фреймворка.  
- Минусы: только те маршруты/страницы, которые вы явно пререндерите (обычно только `/`).

### Вариант B: Переход на Next.js (SSR/SSG)

- Перенести проект на Next.js и рендерить лендинг через `getServerSideProps` или `getStaticProps`.
- Плюсы: полный контроль над HTML, идеально для поисковиков и ИИ.  
- Минусы: миграция, другой деплой и настройка.

### Вариант C: Оставить как есть

- Google и современные боты выполняют JS и часто «видят» SPA нормально.
- ИИ-краулеры и часть ботов могут видеть только HTML; для них уже полезны метатеги и `<noscript>` из `index.html`.

Рекомендация: если не планируете миграцию на Next.js — добавить **пререндер главной** (вариант A) и оставить текущие метатеги и `robots.txt`/`sitemap.xml`.
